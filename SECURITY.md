# Security policy

## Supported versions

Until versioned releases are published, security fixes are applied to the
latest commit on `main` only.

## Reporting a vulnerability

Do not open a public issue for a suspected vulnerability, exposed credential,
or bypass of an identity, routing, preview, permission, or redaction boundary.

Use **Report a vulnerability** from the repository's **Security** tab to send a
private report. Include the affected version or commit, reproduction steps,
impact, and any suggested mitigation. If private vulnerability reporting is
not available, contact the maintainer privately through their GitHub profile
before disclosing details.

Please allow time to confirm the report and coordinate a fix before public
disclosure. Never include live credentials or personal project configuration
in a report; use redacted or temporary test values.
